## 1. 为什么子 Agent 的提示词格外关键

子 Agent 只能看到主代理传给它的那段 prompt，不知道整体任务背景。prompt 模糊会导致：

- 自己脑补意图 → 方向跑偏
- 返回格式不对 → 主代理解析失败
- 做多或做少 → 影响后续所有 Agent

主代理在一个高上下文环境里推理，而子 Agent 是"从零起步"。**写给子 Agent 的 prompt 必须自包含。**

---

## 2. 提示词设计结构

```
1. 角色定义      → 你是谁，负责什么
2. 任务描述      → 这次具体要做什么
3. 输入说明      → 给你的数据是什么格式
4. 约束条件      → 不能做什么，边界在哪
5. 输出格式      → 返回什么，格式是什么
6. 示例（可选）  → 给一个输入输出的例子
```

---

## 3. 示例对比

差的 prompt：
```
帮我分析这段代码有没有问题
```

好的 prompt：
```
你是一个代码审查专家，只负责检查安全漏洞。

输入：一段 Python 函数代码
任务：识别其中的 SQL 注入、XSS、敏感信息泄露风险
约束：不要优化性能，不要修改代码，只做安全分析
输出格式：
  - 问题列表（每条包含：位置、风险类型、严重程度）
  - 如无问题，返回"无安全风险"
```

---

## 4. 核心设计原则

| 原则 | 说明 |
|------|------|
| 单一职责 | 一个子 Agent 只做一件事 |
| 输出格式强约束 | 明确告诉它返回什么结构 |
| 边界清晰 | 明确说不做什么 |
| 上下文最小化 | 只给它完成任务必要的信息 |

---

## 5. 小结：写给子 Agent 的 prompt 和写给人的 prompt 有什么不同

| 维度 | 写给人 | 写给子 Agent |
|------|--------|-------------|
| 背景信息 | 可以省略，人能自己问 | 必须写全，它问不了 |
| 输出格式 | 可以口述 | 必须结构化，主代理会解析 |
| 边界说明 | 可以模糊 | 必须明确，否则它就会跑偏 |
| 示例样本 | 可选 | 强烈推荐，特别是格式复杂时 |

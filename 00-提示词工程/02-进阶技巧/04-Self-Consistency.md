# Self-Consistency（自洽性投票）

> **同一问题多次采样，对答案投票取多数。** 用"多次独立思考 + 集成"来对冲 LLM 单次采样的随机性。

---

## 1. 核心思想

LLM 在 temperature > 0 时每次输出可能不同。  
对于**有唯一正确答案**的任务，多条推理链中**正确答案出现的频次往往高于任何单一错误答案**。

```
采样 5 次：
链1 → 答案 A
链2 → 答案 A
链3 → 答案 B
链4 → 答案 A
链5 → 答案 C

多数投票 → A（3/5）
```

前提假设：**错误分布是发散的，正确答案是收敛的**。

---

## 2. 使用流程

```
1. 写一个带 CoT 的 Prompt
2. 同一 Prompt，用 temperature = 0.7~1.0 调用模型 N 次（典型 N = 5~20）
3. 从每次输出中抽取最终答案
4. 对答案计数，返回 top-1
```

关键点：
- **必须开温度**。T=0 采样退化为单次调用，失去意义
- **必须有 CoT**。只抽短答案投票，错误之间也会收敛，失去对冲效果
- **抽取答案要鲁棒**。推理链后加结构化标签如 `Final Answer: X` 或 `<answer>X</answer>`

---

## 3. 适用场景

| 场景 | 效果 |
|------|------|
| 数学计算、逻辑推理 | ✅ 准确率明显提升（论文常报 +5~15%） |
| 多选题、闭合分类 | ✅ 答案空间小，投票稳定 |
| 事实抽取（有标准答案） | ✅ 对冲幻觉 |
| **开放式生成**（写文章、翻译） | ❌ 答案空间无限，无法投票 |
| **代码生成** | ⚠️ 简单片段可行（按 AST 等价归类），复杂代码不适用 |

---

## 4. 投票策略

### 4.1 精确匹配
数字、枚举值直接字符串相等。

### 4.2 归一化后匹配
- 数学：`2.0` / `2` / `两` 视为等价 → 解析成数字再比
- 文本：去空格、小写、同义词归一

### 4.3 语义聚类
开放式短答案用 embedding 聚类，取最大簇。成本高，慎用。

### 4.4 加权投票
若能给每条链打"自信度分数"（比如链长、是否包含关键步骤），按权重投票。

---

## 5. 成本权衡

Self-Consistency 的代价是 **N 倍 token 消耗**。

| 问题价值 | 推荐 N |
|---------|--------|
| 高频次、低单次价值 | 1（不用 SC） |
| 关键决策、离线批处理 | 10~20 |
| 评测 / benchmark | 20~40 |

替代方案：
- **Verifier 模式**：一次生成 + 一次让模型检查，比 SC 成本低
- **Best-of-N with Reward Model**：用独立打分模型挑最好的一条，而不是投票

---

## 6. 与其他技巧组合

| 组合 | 备注 |
|------|------|
| CoT + Self-Consistency | 原始论文配方，强烈推荐 |
| Few-shot CoT + SC | 稳定性更高，示例质量是上限 |
| ReAct + SC | 少见。多轮工具调用各自不同路径，投票对象应是最终答案而非动作序列 |
| Tree-of-Thoughts + SC | ToT 内部已有搜索+剪枝，叠 SC 收益递减 |

---

## 7. 实现骨架（伪代码）

```python
def self_consistency(prompt, n=10, temperature=0.8):
    answers = []
    for _ in range(n):
        output = llm.call(prompt, temperature=temperature)
        ans = extract_final_answer(output)   # 正则 or 标签解析
        answers.append(normalize(ans))        # 归一化
    return Counter(answers).most_common(1)[0][0]
```

注意：`n` 次调用可以**并行**，延迟不累加。

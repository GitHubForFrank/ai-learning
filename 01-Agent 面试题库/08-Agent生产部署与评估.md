# 08 — Agent 生产部署与评估

---

## ★☆☆ 基础概念

### Q1: Agent 上线前需要做哪些准备？

1. **功能验证**：核心场景的端到端测试通过
2. **安全审查**：输入过滤、输出审核、权限校验就位
3. **性能测试**：延迟、并发、资源消耗达标
4. **成本估算**：Token 消耗预算与实际偏差在可控范围
5. **可观测性**：日志、指标、链路追踪三大支柱就位
6. **回滚方案**：出现问题时的应急回滚路径

### Q2: Agent 的可观测性三大支柱是什么？

| 支柱 | 内容 | 用途 |
|------|------|------|
| 日志 | Agent 推理链路、工具调用、错误详情 | 问题排查 |
| 指标 | 延迟/成功率/Token消耗/调用频次 | 性能监控与告警 |
| 链路追踪 | 跨 Agent 调用链关联（trace_id） | 端到端定位 |

### Q3: 什么是 Token 消耗优化？

在不影响输出质量的前提下降低 API 调用成本：

1. **模型路由**：简单任务用小模型，复杂任务用大模型
2. **语义缓存**：相似问题命中缓存直接返回
3. **上下文压缩**：只注入必要的检索片段
4. **Prompt 精简**：去掉冗余描述，只保留必要指令
5. **批量处理**：合并多个小请求为一次调用

## ★★☆ 应用场景

### Q4: Agent 的评估体系应包含哪些指标？

| 维度 | 指标 | 说明 |
|------|------|------|
| 质量 | 准确率、召回率、答案一致性 | 输出是否正确 |
| 效率 | 端到端延迟（P50/P95/P99） | 响应速度 |
| 成本 | Token 消耗/次、日均费用 | 经济性 |
| 可靠性 | 成功率、可用性（SLO） | 稳定性 |
| 安全 | 护栏拦截率、PII 泄露率 | 安全性 |
| 体验 | 用户满意度、任务完成率 | 用户感知 |

### Q5: RAG 系统上线前的离线评估怎么做？

1. **构建评估集**：500+ 标注问答，覆盖标准/边界/对抗三类
2. **评估指标**：
   - 召回率（Recall@K）：相关文档是否被检索到
   - 命中率：Top-K 中是否包含正确答案
   - 答案一致性：答案与检索片段的语义对齐度
3. **回滚阈值**：一致性 < 90% 不放量
4. **上线后监控**：日采样 100 条人工标注做漂移监控

### Q6: 如何做 Agent 的 A/B 测试？

1. **定义指标**：明确评估维度（准确率、延迟、用户满意度）
2. **流量分割**：10% 流量走新版本，90% 走旧版本
3. **数据收集**：记录两组的关键指标
4. **统计分析**：显著性检验（新版本是否显著优于旧版本）
5. **决策**：达标 → 全量；不达标 → 回滚分析

### Q7: Agent 的延迟优化有哪些手段？

1. **流式输出**：边生成边返回，减少用户感知延迟
2. **并行调用**：独立工具调用并行执行
3. **缓存预热**：高频检索结果预加载
4. **模型蒸馏**：用小模型替代大模型处理简单场景
5. **超时熔断**：单个工具调用设置超时（如 800ms），超时则降级

### Q8: 如何设计 Agent 的语义缓存？

```
用户提问
  → 计算提问的语义向量
  → 在缓存库中搜索相似问题（相似度 > 0.95）
  → 命中：直接返回缓存答案
  → 未命中：正常推理 + 将结果写入缓存
```

注意事项：
- 缓存失效策略（知识更新后缓存作废）
- 非确定性问题的缓存（需要具体实时信息的不能缓存）
- 多租户场景下的缓存隔离

### Q9: Agent 系统的故障模式和应对策略？

| 故障模式 | 表现 | 应对 |
|----------|------|------|
| LLM 不可用 | 5xx 错误 | 重试 + 降级模型 + 熔断 |
| 工具超时 | 单次调用卡住 | 设置超时 + 重试 + 降级 |
| Token 超限 | 上下文溢出 | 自动压缩 + 截断 |
| 质量下降 | 输出变差 | 回滚 Prompt + 告警 |
| 成本飙升 | 调用量/Tokens 异常 | 限流 + 告警 |

### Q11: 如何实现模型路由以优化成本和质量？

```
用户提问
  → 分类器：判断问题复杂度
  → 问候/澄清类 → 小模型（省钱）
  → 标准业务类 → 中模型（平衡）
  → 复杂推理/低置信度 → 大模型（质量优先）
```

配套策略：
- 缓存命中直接返回，不走模型
- 小模型置信度低时升级到大模型
- 按场景预设路由规则，定期复盘调整

### Q12: 建立 Agent 系统的持续改进循环

```
上线 → 监控 → 发现问题 → 分析根因 → 改进 → 验证 → 再上线
```

具体机制：
1. **日采样评估**：每天抽取 100 条生产对话人工标注
2. **漂移检测**：监控关键指标的趋势变化
3. **反馈闭环**：用户反馈 → 分类归因 → 定向改进
4. **回归防护网**：每次改进后跑离线评估集，防止退化
5. **迭代节奏**：安全/质量问题即时修复，体验问题按周迭代

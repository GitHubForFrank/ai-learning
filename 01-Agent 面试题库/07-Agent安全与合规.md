# 07 — Agent 安全与合规

---

## ★☆☆ 基础概念

### Q1: Agent 系统面临哪些主要安全风险？

| 风险类型 | 说明 |
|----------|------|
| Prompt Injection | 恶意输入诱导模型越权操作 |
| 数据泄露 | PII/敏感信息通过模型输出泄露 |
| 幻觉风险 | 模型生成虚假但看似可信的内容 |
| 越权操作 | Agent 执行了超出权限的工具调用 |
| 模型滥用 | 用于生成恶意代码、钓鱼内容等 |
| 供应链风险 | 依赖的第三方插件/工具存在漏洞 |

### Q2: 什么是 Prompt Injection？如何防护？

Prompt Injection 是攻击者通过精心构造的输入，覆盖或绕过系统提示词中的安全约束。

防护措施：
1. **输入过滤**：检测并清理可疑的指令模式
2. **输入输出隔离**：用户输入与系统指令分层，使用特殊分隔符
3. **权限最小化**：Agent 不应拥有超出必要范围的工具权限
4. **输出审核**：对 Agent 生成的输出进行安全检查
5. **人工确认**：高风险操作前请求人工确认

### Q3: 什么是 AI 幻觉（Hallucination）？如何检测？

AI 幻觉是指模型生成的内容看似合理但实际上不真实、不准确或完全虚构。

检测方法：
1. **事实校验**：将输出与检索证据进行语义对齐打分（如 ≥ 0.85 才放行）
2. **溯源验证**：要求输出附带引用来源
3. **多路交叉验证**：用不同模型/工具验证同一事实
4. **置信度标注**：低置信度时标注"待验证"
5. **知识库对比**：与已知事实库做匹配检查

### Q4: Agent 系统中 PII（个人身份信息）如何处理？

1. **入库前脱敏**：姓名→姓氏+*、保单号→后4位、敏感字段移除
2. **传输加密**：PII 在 API 调用中加密传输
3. **输出再扫**：模型输出侧检测 PII，发现则拦截或脱敏
4. **访问控制**：PII 仅授权角色可见
5. **审计日志**：所有 PII 访问记录留痕

## ★★☆ 应用场景

### Q5: Agent 安全护栏（Guardrails）体系包含哪些层？

```
输入层
  ├─ 输入过滤（敏感词、注入检测）
  ├─ 请求体长度限制
  └─ 限流（防止滥用）

推理层
  ├─ 权限校验（工具调用前）
  ├─ 参数校验（格式化与范围检查）
  └─ 上下文隔离（多租户）

输出层
  ├─ 内容审核（有害内容检测）
  ├─ PII 再扫描
  ├─ 幻觉检测
  └─ 格式校验
```

### Q6: 多租户场景下如何确保数据隔离？

三层隔离：
1. **向量库层**：租户 ID 作为过滤条件，索引分区
2. **知识图谱层**：图谱节点标记租户归属
3. **缓存层**：缓存 key 带租户前缀

遇到无法确认隔离边界的需求应先停下澄清，避免上线未隔离的共享索引。

### Q7: Agent 的安全审计应覆盖哪些维度？

1. **代码安全**：SQL 注入、XSS、敏感信息泄露、反序列化漏洞
2. **网络架构安全**：加密传输、访问控制、网络隔离
3. **系统配置安全**：最小权限、补丁管理、安全基线
4. **数据安全**：加密存储、脱敏、备份
5. **合规检查**：等保/GDPR/SOC2 逐条对照

### Q8: Agent 输出如何做到可溯源？

溯源链要求：
- 保留检索片段（哪些文档被召回）
- 保留工具调用参数（调了什么、传了什么）
- 保留提示模板版本（用了哪个版本的 prompt）
- 保留中间步骤（推理链路完整记录）

使任一输出可定位到证据与决策路径。

### Q9: 安全审计报告中漏洞分级标准？

| 等级 | 标准 | 响应时间 |
|------|------|---------|
| P0（严重） | 可被远程利用 | 2 小时内升级 |
| P1（高） | 需认证但影响大 | 24 小时内 |
| P2（中） | 需特定条件 | 下个迭代 |
| P3（低） | 建议修复 | 排期修复 |

配合 CVSS 评分量化风险，每条漏洞附复现步骤与修复建议。

## ★★★ 架构设计

### Q10: 设计一个 Agent 安全防护体系

```
┌─────────────────────────────────────┐
│          安全策略管理层               │
│  权限矩阵 | 审计日志 | 合规检查       │
├─────────────────────────────────────┤
│          安全执行层                   │
│  输入护栏 | 输出审核 | 幻觉检测       │
├─────────────────────────────────────┤
│          安全基础设施层               │
│  加密 | 脱敏 | 限流 | WAF | 鉴权     │
└─────────────────────────────────────┘
```

关键设计原则：
- 默认拒绝，显式授权
- 深度防御（多层拦截）
- 最小权限原则
- 所有操作可审计

### Q11: Agent 直接参与高风险决策（金额、医疗、法务）需要哪些额外保障？

1. **人工最终审核**：模型输出不作为最终决策依据
2. **置信度阈值**：低于阈值自动升级人工处理
3. **多模型交叉验证**：不同模型独立分析，结果不一致时升级
4. **完整溯源链**：每一条建议可追溯到证据来源
5. **责任归属**：明确 AI 辅助角色，最终决策责任在人

原则：模型输出不直接驱动金额、信贷、医疗等高风险决策。

### Q12: 如何建立 Agent 系统的安全响应机制？

1. **监控告警**：异常模式检测（高频调用、异常参数、敏感操作）
2. **分级响应**：按严重度定义响应时间（P0: 2h, P1: 24h, P2: 下迭代）
3. **应急处置**：一键下线问题 Agent/工具的能力
4. **事后复盘**：根因分析 + 回归测试用例补充
5. **持续更新**：根据新型攻击模式更新防护规则

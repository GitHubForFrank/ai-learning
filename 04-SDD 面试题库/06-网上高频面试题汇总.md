# 06 — 网上高频面试题汇总

> 精选 2024-2025 大厂高频面试题（字节/阿里/腾讯/百度/智谱等），覆盖 Agent / LLM / RAG / 多模态 / 工程化

## 一、LLM 基础（⭐）

### Q1：Transformer 的核心架构？Self-Attention 公式？
**参考答案：**
Transformer 由 Encoder + Decoder 组成，核心是 Self-Attention 机制。

$$Attention(Q,K,V) = softmax(\frac{QK^T}{\sqrt{d_k}})V$$

**关键概念**：Q（Query）/ K（Key）/ V（Value）、Multi-Head Attention、位置编码、残差连接+LayerNorm

### Q2：LLM 的训练流程（Pre-training → SFT → RLHF）？
**参考答案：**
1. **Pre-training**：海量无标注数据，自回归语言建模，学习通用知识
2. **SFT（Supervised Fine-Tuning）**：人工标注的高质量指令-回答对，学会遵循指令
3. **RLHF（Reinforcement Learning from Human Feedback）**：人类偏好训练奖励模型，PPO 强化学习对齐人类偏好

**DPO 替代 RLHF**：直接偏好优化，更稳定、更简单。

### Q3：Tokenizer / Tokenization 的常见算法？
**参考答案：**
- **BPE（Byte Pair Encoding）**：GPT 系列常用，从字符开始逐步合并高频对
- **WordPiece**：BERT 使用，基于概率而非频率
- **SentencePiece**：T5/LLaMA 使用，直接在字节层面处理，不依赖预分词

**关键概念**：词表大小、OOV 处理、分词粒度（token 越多=越贵、越细=越准）

### Q4：什么是 Context Window？长上下文的挑战？
**参考答案：**
- Context Window 是 LLM 一次能处理的最大 token 数
- 挑战：注意力矩阵 O(n²) 增长、位置编码外推、长程信息丢失
- 优化：RoPE 扩展、Sliding Window Attention、Ring Attention、Prompt Caching

## 二、Agent 核心技术（⭐⭐）

### Q5：Agent 的 Planning 模块有哪些主流方法？
**参考答案：**

| 方法 | 原理 | 代表 |
|------|------|------|
| ReAct | 推理+行动交替 | ReAct Paper |
| Plan-and-Execute | 先完整计划再执行 | Plan-and-Solve |
| Tree-of-Thought | 树状搜索多路径 | ToT Paper |
| LLMCompiler | 并行执行+DAG 依赖 | LLMCompiler |
| AutoGPT | ReAct+记忆+子目标 | AutoGPT |

**选型**：任务确定→Plan-Execute；探索性→ReAct；需要多路径比较→ToT；需要并行→LLMCompiler。

### Q6：LangChain 的核心组件？为什么有人说 LangChain "抽象泄漏"？
**参考答案：**
核心组件：Chains / Agents / Tools / Memory / Retrievers / Callbacks

"抽象泄漏"争议：
- 过度封装导致调试困难（不知道 LLM 实际发了什么 prompt）
- 版本迭代快、API 不稳定
- 对简单场景过于复杂，对复杂场景又不够灵活
- 2024 年后趋势：越来越多团队选择手写编排或用轻量框架

### Q7：System Prompt 设计的最佳实践？
**参考答案：**
6 要素结构：
1. **角色定义**：你是什么角色/专家
2. **任务描述**：具体要做什么
3. **约束/红线**：绝对不能做什么
4. **输出格式**：期望的输出结构
5. **工作流程**：分步骤指导
6. **示例**：Few-shot 示例

SDD 下的扩展：AGENTS.md = 项目级 System Prompt，角色 agent prompt = 阶段级 System Prompt。

### Q8：Tool Calling 的可靠性如何保障？
**参考答案：**
1. **Schema 设计**：参数类型精确、enum 约束、description 清晰
2. **错误处理**：重试机制（exponential backoff）、降级策略、换工具
3. **参数校验**：调用前做 Schema validation、调用后做 Response validation
4. **安全边界**：只暴露必要工具、读写权限分离、白名单机制
5. **Fallback**：工具不可用时提供纯文本替代方案

## 三、RAG（检索增强生成）（⭐⭐⭐）

### Q9：RAG 的完整 Pipeline 是什么？
**参考答案：**
```
文档 → 分块(Chunking) → 向量化(Embedding) → 向量数据库存储
                                            ↓
用户 Query → 向量化 → 检索(Retrieval) → 重排序(Reranking)
                                            ↓
                    LLM 生成（Query + 检索结果作为上下文）→ 最终答案
```

### Q10：RAG 的核心挑战和优化策略？
**参考答案：**

| 挑战 | 优化策略 |
|------|---------|
| 分块粒度不当 | 语义分块、Small-to-Big 检索 |
| 检索噪声 | Hybrid Search（向量+关键词）、Reranking |
| Query-Document Gap | HyDE（假设文档嵌入）、Query Rewriting |
| 检索不完整 | 多轮检索、Self-RAG |
| 答案质量 | 引用溯源、Faithfulness 校验 |

### Q11：GraphRAG 和传统 RAG 的区别？
**参考答案：**
- **传统 RAG**：向量相似度检索，擅长语义匹配，弱于多跳推理
- **GraphRAG**：构建知识图谱，检索实体+关系，擅长全局总结和多跳推理
- **选择**：简单问答→传统 RAG；需要理解全局结构/多实体关联→GraphRAG

## 四、多 Agent 系统（⭐⭐⭐）

### Q12：多 Agent 系统的常见框架对比？
**参考答案：**

| 框架 | 特点 | 适合 |
|------|------|------|
| CrewAI | 角色定义清晰、串行/并行编排 | 中等复杂度 |
| AutoGen (Microsoft) | 对话驱动、灵活度高 | 研究/探索 |
| LangGraph | 图结构编排、状态管理强 | 复杂流程 |
| MetaGPT | 模拟软件公司 SOP | 软件开发全流程 |
| Swarm (OpenAI) | 轻量级、handoff 模式 | 简单路由+切换 |

### Q13：多 Agent 的任务分配策略有哪些？
**参考答案：**
1. **按专长分配**：每个 Agent 有专门的能力描述，Router Agent 按任务类型分发
2. **协商分配**：Agent 之间广播任务，能力匹配者竞标
3. **层级分配**：Manager Agent 分解任务，分派给 Worker Agent
4. **投票/裁决**：多个 Agent 给出方案，裁决者选择最优

### Q14：多 Agent 协作中的冲突如何解决？
**参考答案：**
- **去重**：多个 Agent 产出相似结果→去重合入
- **投票**：不同意见→多数决或加权投票（专家权重高）
- **裁决者**：冲突上升给专门的仲裁 Agent
- **人类介入**：无法自动裁决→人工判断
- **SDD 视角**：以 spec 为准——spec 说得很清楚的不允许冲突

## 五、工程化与性能优化（⭐⭐）

### Q15：大模型推理的常见优化技术？
**参考答案：**
- **量化（Quantization）**：FP16→INT8/INT4，降低显存和延迟
- **KV Cache**：缓存已计算的 Key-Value 对，加速自回归生成
- **Continuous Batching**：动态合并请求，提高 GPU 利用率
- **Speculative Decoding**：小模型草稿+大模型验证，加速生成
- **Flash Attention**：优化注意力计算的内存访问模式

### Q16：Agent 服务的延迟优化策略？
**参考答案：**
- 模型蒸馏（小模型替代部分调用）
- 异步处理（非实时任务走队列）
- 工具调用缓存（相同查询不重复调 API）
- 并行工具调用（无依赖的工具同时调）
- Prompt Caching（缓存 system prompt 和稳定上下文）

### Q17：Agent 系统的可观测性如何设计？
**参考答案：**
- **Metrics**：成功率、延迟、Token 消耗、工具调用次数
- **Logs**：每次 LLM 调用的完整 prompt/response、工具调用记录
- **Traces**：单个任务从接收到完成的完整执行链路
- **告警**：成功率下降/延迟飙升/费用异常→触发通知

## 六、安全与合规（⭐⭐）

### Q18：Agent 的安全风险及防范措施？
**参考答案：**

| 风险 | 防范 |
|------|------|
| Prompt Injection | 输入过滤、指令隔离、角色强化 |
| 工具滥用 | 最小权限、白名单、人工审批 |
| 数据泄漏 | 敏感信息脱敏、不写入训练数据、最小数据原则 |
| 幻觉/误操作 | RAG 增强、人工审核节点、操作回滚 |
| 资源滥用 | 限流（速率限制）、配额管理 |

### Q19：如何评估一个 RAG 系统的质量？
**参考答案：**
常用评估框架 **RAGAS** 的指标：
- **Faithfulness**：生成内容是否与检索文档一致
- **Answer Relevance**：回答是否与问题相关
- **Context Recall**：检索是否覆盖了回答所需的全部信息
- **Context Precision**：检索到的文档中相关比例

## 七、2025 热点（⭐）

### Q20：MCP 协议对未来 Agent 生态的影响？
**参考答案：**
- **标准化**：统一工具调用接口，降低 Agent↔工具集成的 M×N 复杂度
- **生态效应**：类似"App Store for AI"，工具开发者写 MCP Server 即可触达所有 Agent
- **安全边界**：通过 Server 层白名单而非 Prompt 层做安全约束
- **SDD 中的角色**：MCP 让 Agent 直接查 DB/日志/API，加速开发期调试

### Q21：从单 Agent 到 Multi-Agent 系统的核心挑战？
**参考答案：**
1. **通信协议**：Agent 间用什么格式交换信息
2. **上下文管理**：多个 Agent 的上下文如何不互相污染
3. **任务分配**：如何避免重复工作和任务遗漏
4. **冲突解决**：不同 Agent 产生矛盾结果时怎么办
5. **成本控制**：多 Agent 意味着多倍 LLM 调用，如何控制在预算内
6. **失败传播**：一个 Agent 挂了会如何影响其他 Agent

### Q22：2025 年 AI Agent 面试最可能被问到的开放题？
**参考答案：**
1. 设计一个能帮开发者写代码+测试+部署的 AI Agent 系统
2. 多 Agent 系统中如何实现"只读 spec"的机制
3. 你如何看待 SDD 是"瀑布模型的回潮"
4. Agent 的"自主性"应该在什么级别（任务级/子任务级/操作级）
5. 如果让你从零搭一个 Agent 团队，你会怎么分配职责和选模型

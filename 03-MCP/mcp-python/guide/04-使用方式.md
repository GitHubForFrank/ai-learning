## 1. 传输层配置

### 1.1 Stdio（默认）

```python
await mcp.run_stdio_async()
```

### 1.2 Streamable HTTP

```python
await mcp.run_streamable_http_async(host="0.0.0.0", port=8000)
```

### 1.3 命令行切换

```python
# __main__.py
import argparse

parser = argparse.ArgumentParser()
parser.add_argument("--transport", choices=["stdio", "http"], default="stdio")
parser.add_argument("--port", type=int, default=8000)
args = parser.parse_args()

if args.transport == "stdio":
    asyncio.run(mcp.run_stdio_async())
else:
    asyncio.run(mcp.run_streamable_http_async(port=args.port))
```

## 2. 调试

### 2.1 使用 MCP Inspector

```bash
# 必须在项目根目录（含 pyproject.toml）下，uv run 才能命中正确的 .venv
cd /path/to/my-mcp-server

npx @modelcontextprotocol/inspector uv run python -m my_mcp_server
```

终端会打印 Inspector 的本地 URL（形如 `http://localhost:6274`），浏览器打开即可：
- 查看 Tool/Resource/Prompt 列表
- 手动调用 Tool 并查看响应
- 查看原始 JSON-RPC 消息

### 2.2 单元测试

```python
# tests/test_server.py
import pytest
from my_mcp_server.entrypoints.mcp import mcp     # FastMCP 实例定义在 entrypoints/mcp.py

@pytest.mark.asyncio
async def test_get_weather():
    async with mcp.session() as session:
        result = await session.call_tool("get_weather", {"city": "Beijing"})
        assert "Beijing" in result.content[0].text
```

**运行**：

```bash
# 必须在项目根目录（含 pyproject.toml 与 tests/）下
cd /path/to/my-mcp-server

uv run pytest                             # 全量
uv run pytest tests/test_server.py -v     # 单文件
```

首次运行前，先 `uv add --dev pytest pytest-asyncio` 把测试依赖写进 `pyproject.toml` 的 `[project.optional-dependencies].dev`。

## 3. 常见坑

| 坑 | 说明 |
|----|------|
| `print()` 到 stdout | 破坏 Stdio 协议，一律用 stderr 或 logging |
| 同步阻塞调用 | Server 是异步的，同步 IO 需 `asyncio.to_thread` 包装 |
| docstring 太简略 | LLM 看不懂"什么时候用"，调用时机判断不准 |
| 返回巨大字符串 | 考虑返回 Resource URI 让 Client 按需读取 |
| 未捕获异常 | 会直接抛给 Client，暴露栈信息；关键路径加 try/except |

## 4. 小结

- FastMCP 装饰器风格是 90% 场景的最佳起点
- 用 Pydantic 严格校验参数 → LLM 参数错误能早期拦截
- 区分 `ToolError`（可恢复）与异常（不可恢复）
- stderr 日志 + Inspector 是开发期的主要调试手段
- 打包成 PyPI 包 + uvx 运行 = 最顺滑的分发体验

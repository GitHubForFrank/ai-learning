## 1. 传输层配置

### 7.1 Stdio（本地工具）

使用 `spring-ai-starter-mcp-server` 并在 yml 中：

```yaml
spring:
  ai:
    mcp:
      server:
        protocol: STDIO
        stdio: true
```

**重要**：Stdio 模式下**不要把 log 写到 stdout**，会污染协议通道。

```yaml
logging:
  pattern:
    console: ""       # 关掉 console appender
  file:
    name: mcp-server.log
```

或统一把 log 重定向到 stderr：

```properties
logging.pattern.console=%d{HH:mm:ss} %-5level %logger - %msg%n
spring.main.banner-mode=off
# 程序入口加： System.setOut(System.err);  （极端做法，谨慎）
```

### 7.2 Streamable HTTP

使用 `-webmvc` 或 `-webflux` starter，yml：

```yaml
spring:
  ai:
    mcp:
      server:
        protocol: STREAMABLE
server:
  port: 8080
```

默认端点 `POST /mcp`。

### 7.3 Client 配置（`.mcp.json`）

Stdio：

```jsonc
{
  "mcpServers": {
    "weather": {
      "command": "java",
      "args": ["-jar", "/path/to/my-mcp-server.jar"]
    }
  }
}
```

HTTP：

```jsonc
{
  "mcpServers": {
    "weather": {
      "url": "http://localhost:8080/mcp"
    }
  }
}
```

## 2. 调试

### 8.1 MCP Inspector

```bash
# Stdio 模式
npx @modelcontextprotocol/inspector java -jar target/my-mcp-server.jar

# HTTP 模式（先启动 Server，再打开 Inspector 连接）
npx @modelcontextprotocol/inspector
```

### 8.2 单元测试

用 Spring Boot Test 测试 `@McpTool` 方法本身（当普通 Bean 测）：

```java
@SpringBootTest
class CalculatorToolsTest {
    @Autowired CalculatorTools tools;

    @Test
    void addReturnsSum() {
        assertEquals(5.0, tools.add(2, 3));
    }
}
```

集成测试启动完整 MCP Server 并用官方 MCP Java Client 连接，覆盖协议握手与工具发现。

## 3. 常见坑

| 坑 | 说明 |
|----|------|
| `System.out.println` 污染 stdio | Stdio 模式禁止任何 stdout 输出 |
| Banner 输出到 stdout | 加 `spring.main.banner-mode=off` |
| 启动慢（Stdio） | 考虑 GraalVM Native Image 或换用 HTTP 模式 |
| 注解未被扫描 | 确保 `@Service`/`@Component` 与启动类同包或子包 |
| Jackson 序列化复杂对象失败 | 返回 `String` / `CallToolResult` 自行控制序列化 |
| 依赖冲突 | Spring AI MCP 相关版本需对齐，用 BOM 统一管理 |

## 4. 小结

- **@McpTool 注解式** 是 90% 场景的最佳起点，声明式、零样板
- **`@Tool` + `MethodToolCallbackProvider`** 适合与 Spring AI ChatClient 复用工具
- Stdio 模式要**关闭 console 日志**避免污染协议通道
- HTTP 模式下用 webmvc（传统）或 webflux（响应式）starter
- 冷启动慢是 Java MCP Server 的主要痛点——远程服务不敏感，本地 Stdio 场景考虑 **GraalVM Native**
- 同仓参考实现：`06-MCP 实战项目/mcp-java/`（含 stdio / webmvc / webflux 三个 demo）

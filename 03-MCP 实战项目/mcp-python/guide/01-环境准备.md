## 1. 环境准备

### 1.1 前置：安装 Python 运行时

要求 **Python 3.10+**（SDK 深度使用 `TaskGroup`、类型参数语法等）。

| 平台 | 推荐方式 |
|------|---------|
| Windows | https://www.python.org/downloads/ 官方安装器；或 `winget install Python.Python.3.12` |
| macOS | `brew install python@3.12` |
| Linux | 发行版包管理（`apt install python3.12`）或 [pyenv](https://github.com/pyenv/pyenv) |

**验证**：

```bash
python --version            # 应 ≥ 3.10
# Windows 若 PATH 未配置，用 py launcher：
py --version
```

### 1.2 推荐包管理器：uv

[`uv`](https://docs.astral.sh/uv/)（Rust 编写）是 Python 新一代工具，比 `pip + venv` 快 10 倍，替代 `pip + virtualenv + pip-tools`：

```bash
# Windows
winget install astral-sh.uv
# macOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh
```

**验证**（新开终端执行，不依赖当前目录）：

```bash
uv --version                # 打印类似 `uv 0.4.x`
```

若 `uv: command not found`：Windows 重开终端让 PATH 生效；macOS/Linux 检查 `~/.local/bin` 是否已进 PATH。

无 `uv` 时可退回 `pip + venv`，命令会在下文同时给出。

#### 1.2.1 GUI 客户端挂载 MCP 前：确认 uv 在**系统 PATH**

**为什么要单独讲**：上面 `uv --version` 只验证"终端能找到 uv"。MCP Client（Claude Desktop / Cursor / Cherry Studio / ...）是 GUI 进程，它派生 MCP Server 子进程时继承的是 **用户/系统环境变量**级别的 PATH，**不是**当前终端临时的 `$PATH` / `$env:PATH`。

> 典型翻车：终端里 `uv --version` 正常 → 以为配好了 → 在 Client 的 `.mcp.json` 里 `"command": "uv"` → 客户端起不来 MCP，日志里 `command not found` / `文件未找到`。
>
> Windows 尤其常见——astral 官方的 PowerShell 安装器把 `uv.exe` 装到 `%USERPROFILE%\.local\bin`，这目录默认不在系统 PATH 里。

**诊断**（新开终端；不依赖当前目录）：

```powershell
# Windows PowerShell
where.exe uv
# 无输出 = 系统 PATH 没有 uv —— GUI 客户端也找不到
```

```bash
# macOS / Linux
which uv
# 无输出 = login shell PATH 没有 uv
```

**修复一（推荐）：永久加进用户级 PATH**

Windows（PowerShell，**免管理员**；uv 目录按实际替换，默认是 `$HOME\.local\bin`）：

```powershell
# 不依赖当前目录
[Environment]::SetEnvironmentVariable(
  "PATH",
  [Environment]::GetEnvironmentVariable("PATH", "User") + ";$HOME\.local\bin",
  "User"
)
```

macOS / Linux（追加到登录 shell 启动脚本，具体文件按 `$SHELL` 选）：

```bash
# 不依赖当前目录
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zprofile     # macOS zsh
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.profile       # Linux / bash
```

改完**必须**退出并重启 GUI MCP Client（Windows 极端情况需注销重登桌面会话）——GUI 进程只在启动瞬间读 PATH，热改环境变量对已启动进程无效。

**验证**（新开一个终端 / 新窗口，不要在旧窗口里验，旧窗口 PATH 已缓存）：

```bash
# Windows（cmd / PowerShell 都可）
where.exe uv                    # 应打印 uv.exe 绝对路径

# macOS / Linux
which uv                         # 应打印 uv 绝对路径
```

打印出绝对路径即 GUI 客户端也能找到。

**修复二（临时/绕过）：客户端配置里写 uv 绝对路径**

不想改 PATH，或只在一台机器上短期用，就在 `.mcp.json` / `claude_desktop_config.json` 里把 `command` 写成 uv 的绝对路径，零 PATH 依赖：

```jsonc
{
  "mcpServers": {
    "<server-name>": {
      "command": "C:/Users/<you>/.local/bin/uv.exe",
      "args": ["run", "<package-name>"],
      "cwd": "/path/to/<server-project>"
    }
  }
}
```

权衡：每加一个 MCP Server 都要重写绝对路径；uv 升级换装位置、或换机器，所有配置都要改一遍。**日常优先用修复一**，修复二只作单机临时验证。

### 1.3 从零创建项目

```bash
# 1) 进入你的工作目录
cd D:/workspaces/your-workspace

# 2) 创建项目骨架（uv init 自动生成 pyproject.toml）
uv init my-mcp-server

# 3) 进入项目目录 —— 后续所有 uv 命令都要在这里运行
cd my-mcp-server
```

此时目录结构：

```
my-mcp-server/
├── pyproject.toml
├── README.md
└── main.py                 # 占位，可删
```

### 1.4 安装 MCP SDK

MCP 官方 Python SDK 包名 `mcp`，发布在 **PyPI**：https://pypi.org/project/mcp/

```bash
# 必须在项目根目录（含 pyproject.toml）下运行
uv add mcp

# 或 pip（若未用 uv）
pip install "mcp>=1.0.0"
```

`uv add` 会：
- 把 `mcp` 写入 `pyproject.toml` 的 `[project].dependencies`
- 在 `.venv/` 建虚拟环境并安装
- 生成 `uv.lock` 锁定精确版本

**版本锁定建议**：`uv add "mcp>=1.0,<2.0"`，避免 SDK 大版本跳变破坏代码。

### 1.5 验证安装

```bash
# 必须在项目根目录（含 pyproject.toml）下运行，否则 uv run 会报错或用错环境
cd /path/to/my-mcp-server

uv run python -c "from mcp.server.fastmcp import FastMCP; print('MCP SDK OK')"
```

打印 `MCP SDK OK` 即成功。若报 `ModuleNotFoundError`，检查：
- 当前目录是项目根吗（含 `pyproject.toml`）？
- `.venv/` 存在吗（`uv add` 成功会自动建）？

### 1.6 目标项目结构

最小可用骨架（仅展示主路径；完整目录布局、命名硬约束、各层职责见 [`02-源代码结构.md`](02-源代码结构.md)）：

```
my-mcp-server/
├── pyproject.toml
├── uv.lock
└── src/
    └── my_mcp_server/                # 包名用下划线，与项目名对应
        ├── __init__.py               # 仅暴露 __version__
        ├── __main__.py               # `python -m my_mcp_server` 薄委托
        ├── <engine>.py               # 引擎 / 编排器，按业务语义命名
        └── entrypoints/              # 传输层：每文件一个 main()
            ├── cli.py                #   CLI 入口
            └── mcp.py                #   MCP Server 入口（stdio + http 共用）
```

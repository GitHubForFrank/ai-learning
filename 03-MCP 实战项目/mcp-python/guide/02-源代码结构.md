## 1. 推荐目录骨架

遵循「src 布局 + 双入口 + 引擎/传输分层」：

```
my-mcp-server/
├── README.md
├── pyproject.toml
├── uv.lock
├── development/                      # [软] 开发期资料，不随 wheel 发布
│   ├── plans/                        #   架构迭代方案 PLAN_vNN.md
│   ├── findings/                     #   调研 / 验证产出 FINDINGS_<ts>.md
│   └── samples/                      #   测试样本数据
├── src/
│   └── my_mcp_server/                # 包名用下划线，与项目名 my-mcp-server 对应
│       ├── __init__.py               # [必需] 仅暴露 __version__，不放业务
│       ├── __main__.py               # [必需] python -m 入口，薄委托到 entrypoints.cli
│       ├── <engine>.py               # [必需] 引擎/编排器，按业务语义命名
│       ├── common.py                 # [软] 跨模块共享：路径/日志/配置加载
│       ├── <domain>.py               # [软] 引擎大时按子领域再拆
│       ├── i18n.py                   # [软] 多语言消息
│       ├── entrypoints/              # [必需] 传输层：每文件一个 main()
│       │   ├── __init__.py
│       │   ├── cli.py                #   CLI 入口（argparse + 引擎调用）
│       │   └── mcp.py                #   MCP Server 入口（FastMCP；stdio + http 共用）
│       ├── knowledge/                # [软] 静态知识资源（YAML/JSON），随包分发
│       ├── phases/                   # [软] 多阶段流水线（每阶段一文件）
│       └── tools/                    # [软] 外部工具/二进制（jar、脚本等）
└── tests/                            # [软] pytest + pytest-asyncio
```

`[必需]` / `[软]` 含义：必需违反会破坏分层契约；软项视项目规模裁剪。

## 2. 命名硬约束

违反会与 [`../mcp_develop_rule.md`](../mcp_develop_rule.md) 冲突：

- `<engine>.py` 按**业务语义**命名（如 `analyze.py` / `convert.py` / `summarize.py`），**不用** `server.py` / `core.py` / `engine.py` 这类泛名（规则 §3.3）
- `mcp.py` **一个文件同时支持 stdio + http**，`main()` 内用 `--transport` argparse 分支切换；**不要**拆 `mcp_stdio.py` / `mcp_http.py`（规则 §3.4 拆分粒度红线）
- 包名（`src/<package>/`）用下划线，与 `pyproject.toml [project].name` 的连字符版本一一对应

## 3. 各层职责（一句话）

| 层 | 职责 | 不该做的事 |
|----|------|-----------|
| `pyproject.toml` | name / version / dependencies / `[project.scripts]` 入口注册 | 内联业务逻辑 |
| `src/<pkg>/__init__.py` | 仅暴露 `__version__` | 写实现 |
| `src/<pkg>/__main__.py` | `python -m <pkg>` 薄委托到 `entrypoints.cli:main` | 写业务、注册工具 |
| `src/<pkg>/<engine>.py` | 引擎 / 编排器：纯业务逻辑，不感知 Transport | 引用 `mcp.*` SDK 类型 |
| `src/<pkg>/entrypoints/cli.py` | CLI 入口：argparse + 调引擎方法 + 输出格式化 | 实现业务 |
| `src/<pkg>/entrypoints/mcp.py` | MCP Server 入口：`FastMCP` 实例 + `@mcp.tool` + transport 选择 | 实现业务（薄包装到引擎方法） |

**分层原则**：引擎放父 package，`entrypoints/*.py` 都是薄适配层——把某种**协议/调用模型**翻译为对引擎的调用。新增协议（HTTP 业务 API、gRPC 等）只需新建 `entrypoints/<name>.py` + 在 `[project.scripts]` 注册一行。

## 4. 注册约定

### 4.1 FastMCP 装饰器（推荐）

```
mcp = FastMCP("my-server")

@mcp.tool()
def some_tool(...): ...

@mcp.resource("uri://...")
def some_resource(...): ...

@mcp.prompt()
def some_prompt(...): ...
```

约定：

- `mcp = FastMCP(name)` 实例放在 `entrypoints/mcp.py` 模块顶层
- 工具实现**只做参数校验 + 调引擎方法**，业务代码留在 `<engine>.py`
- 函数 docstring 是 LLM 选工具的关键线索——写法见 [`../mcp_develop_guideline.md`](../mcp_develop_guideline.md)

### 4.2 类型注解 → JSON Schema

FastMCP 通过函数签名 + Pydantic 自动生成 JSON Schema：

| Python 类型 | JSON Schema |
|------------|-------------|
| `str` / `int` / `float` / `bool` | string / integer / number / boolean |
| `Literal["a", "b"]` | enum |
| `pydantic.BaseModel` 子类 | object（字段递归映射） |
| `list[T]` / `dict[K, V]` | array / object |
| 默认值 | 非 required + default |

参数描述用 `pydantic.Field(description="...")` 或 docstring；严格校验在 LLM 给错参数时**早期拦截**。

### 4.3 错误处理

| 方式 | 效果 |
|------|------|
| 抛 `mcp.ToolError` | 包装为 `isError=true`，LLM 可读到原因并纠正 |
| 抛其他异常 | 走 JSON-RPC error，Client 看到协议级错误（暴露栈风险） |

建议：用户可恢复的错误（参数非法、资源不存在）抛 `ToolError`；未预期的内部错误让其自然冒泡前**先 try/except 包成 ToolError**。

## 5. Stdio vs HTTP：源码层差异

差异**只在 `entrypoints/mcp.py` 的 `main()` 一处**，引擎与工具实现完全复用：

| Transport | FastMCP 调用 | 进程模型 |
|-----------|------------|---------|
| stdio | `await mcp.run_stdio_async()` | Client 拉起子进程，stdin/stdout 通信 |
| http | `await mcp.run_streamable_http_async(host=..., port=...)` | 长驻 HTTP 服务，多 Client 共享 |

**强约束**：

- stdio 模式下**所有 `print()` 都会污染 stdout 协议通道**——一律改用 `logging` 写 stderr，或显式 `print(..., file=sys.stderr)`
- 同步阻塞调用要用 `asyncio.to_thread(...)` 包装，否则会卡住整个事件循环
- CLI 通过 `--transport stdio|http` 切换的模板见 [`04-使用方式.md §1.3`](04-使用方式.md)
